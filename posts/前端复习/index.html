HTML

    async异步请求，请求完成后同步执行，可能阻塞页面解析。

CSS

  1. 盒模型

    • content-box 标准盒模型
    • border-box 替代盒模型

    height/width计算范围：
    content-box: content
    border-box: content + padding + border

  2. CSS选择器优先级

    四位评分制：

    A B C D
    内联 ID 类/属性/伪类 标签/伪元素

    从左到右比较，应用大的。

  3. 重排/重绘

    重排：无论什么方式影响了元素在视窗内的位置大小，浏览器需要重新计算元素在视窗内的位置大小，这个过程叫重排。
    重绘：将重排后在视窗中可见元素，结合元素样式，大小和位置绘制在屏幕上的过程叫重绘。

  4. 如何减少重排重绘？

    最小化重排重绘：样式集中改变，比如class。
    批量操作DOM：比如使用document.createFragment来添加节点，处理完成后插入到实际DOM中。
    减少影响元素：使用absoulte/fixed使元素脱离文档流。
    使用GPU加速：使用transform/will-change等，使用GPU绘制。

  5. BFC

    块级格式上下文。

    BFC特性：

    • BFC内元素会发送外边距重叠
    • 计算BFC高度，需要计算其中浮动元素高度
    • BFC不会与浮动元素重叠
    • BFC内部元素不会影响外部元素

    怎样创建BFC：

    • display: inline-block
    • position: absolute/fixed
    • overflow不为visible
    • float不为none
    • display: flex(FFC)
    • display: grid(GFC)

HTTP

  1. 缓存

    不缓存：Cache-Control: no-store

    强制缓存：没有Pragma: no-cache、Cache-Control: no-cache/max-age、Expire时直接使用强制缓存

    协商缓存：有上面属性时需要与服务器校验缓存是否过期，使用ETag/If-Not-Match、Last-Modified/If-Modified-Since判断。

    如果服务器校验未过期，返回304，使用本地缓存，否则返回200和新数据。

  2. HTTP/2

    HTTP/1.1限制：

    最多并行请求数限制。

    特性：

    • 二进制分帧层
    • 流量控制
    • 头部压缩
    • 多路复用
    • 服务器推送
    • 优先级控制

    原理：

    创建一个TCP连接，在连接上创建多个流，在流上传送数据帧。

  3. loader和plugin原理

    Loader 本质就是一个函数，在该函数中对接收到的内容进行转换，返回转换后的结果。
    因为 Webpack 只认识 JavaScript，所以 Loader 就成了翻译官，对其他类型的资源进行转译的预处理工作。
    Plugin 就是插件，基于事件流框架 Tapable，插件可以扩展 Webpack 的功能，在 Webpack 运行的生命周期中会广播出许多事件，Plugin 可以监听这些事件，在合适的时机通过 Webpack 提供的 API 改变输出结果。

Vue

  1. 生命周期及生命周期中执行的方法

    hook desc
    beforeCreate 组件实例化之前，data和methods数据都还未创建
    created 实例创建完成，但是真实dom还没有创建，$el不可用
    beforeMount render函数执行，但节点还未挂载
    mounted el被$el元素替换并挂载
    beforeUpdate 组件数据更新之前调用，发生在虚拟DOM打补丁之前
    update 组件数据更新之后
    beforeDestory 销毁前调用
    destoryed 销毁后调用

  2. 父子组件生命周期

    加载渲染过程
    父 beforeCreate -> 父 created -> 父 beforeMount -> 子 beforeCreate -> 子 created -> 子 beforeMount -> 子 mounted -> 父 mounted

    子组件更新过程
    父 beforeUpdate -> 子 beforeUpdate -> 子 updated -> 父 updated

    父组件更新过程
    父 beforeUpdate -> 父 updated

    销毁过程
    父 beforeDestroy -> 子 beforeDestroy -> 子 destroyed -> 父 destroyed

  3. 虚拟DOM, diff与patch

    虚拟DOM就是将实际的DOM抽象成一个js对象，在vue中，vue将template/render构建成一个虚拟DOM，当数据改变后，会进行新旧虚拟DOM树的对比。
    对比的原则是深度优先，同级比较，当发现某个节点需要被替换时，会将它及子节点全部删除，不会再遍历其子节点。对比同级时，如果tagName不同直接重新生成替换，其他不同可能修改属性，可能调换顺序，key的作用就是为了更方便对比节点是否一致。
    当对比完毕后生成一个diff结果树，会使用老的DOM树与diff结果生成一个新的DOM树。
    然后整体替换，从而减少重排重绘次数。

    虚拟DOM的优点：

    • 批量更新，减少重排重绘。
    • 抹平api差异，兼容性好。

  4. vue双向绑定的实现

    1. 创建订阅器Dep
    2. 使用Object.defineProperty为数据创建响应性，劫持set方法，为set添加监听者Observer，绑定一个Dep
    3. 编译模板时根据绑定的数据创建观察者Watcher，将Watcher添加到对应的Dep中
    4. 修改数据时触发set，通知所有的Watcher更新View

  5. nextTick

    数据更新时，vue先检查待更新队列中watcher的唯一性并将其推入队列中，推入完成后执行nextTick(flushQueue)，将队列放入微/宏任务队列中更新，在执行前会对watcher进行排序，子组件比父组件先执行。

    vue中的nextTick使用多层判断当前环境兼容性，将任务放入微任务/宏任务中执行，分别使用Promise/MutationObserver/setImmediate/setTimeout实现。

    在事件循环一个循环结束后，检查微任务队列，如果微任务队列中有任务，会将其全部执行，再检查宏任务队列，如果有任务，会取出一个执行，然后进入下一个事件循环。
